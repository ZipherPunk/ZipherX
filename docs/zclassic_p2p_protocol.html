// OVERVIEW

This document specifies the exact binary wire format for Zclassic/Zcash Sapling v4 transactions as transmitted over the P2P network. The goal is to enable ZipherX wallet to parse shielded transaction data (vShieldedSpend, vShieldedOutput) from raw P2P block messages.

// BLOCK FORMAT (P2P Wire Protocol)

CBlockHeader (140 bytes fixed)

OffsetSizeFieldDescription
04nVersionBlock version
432hashPrevBlockPrevious block hash
3632hashMerkleRootMerkle root of transactions
6832hashFinalSaplingRootTHIS IS THE SAPLING ANCHOR!
1004nTimeUnix timestamp
1044nBitsDifficulty target
10832nNonceEquihash nonce

After Header

[CompactSize]  Equihash solution length (~1344 bytes typical)
[variable]     Equihash solution data
[CompactSize]  Transaction count
[variable]     Transaction[0], Transaction[1], ...

// SAPLING V4 TRANSACTION FORMAT

Identification: header = 0x04000080 (v4, fOverwintered=true)
nVersionGroupId = 0x892F2085 (SAPLING_VERSION_GROUP_ID)

Transaction Structure

[4 bytes]     header (version + fOverwintered flag)
[4 bytes]     nVersionGroupId (if overwintered)
[CompactSize] vin count
[variable]    vin[0], vin[1], ... (CTxIn)
[CompactSize] vout count
[variable]    vout[0], vout[1], ... (CTxOut)
[4 bytes]     nLockTime
[4 bytes]     nExpiryHeight
[8 bytes]     valueBalance
[CompactSize] vShieldedSpend count
[384 × N]     SpendDescription[0..N-1]
[CompactSize] vShieldedOutput count
[948 × M]     OutputDescription[0..M-1]
[CompactSize] vjoinsplit count
[variable]    JSDescription (if any)
[32 bytes]   joinSplitPubKey (if jsCount > 0)
[64 bytes]   joinSplitSig (if jsCount > 0)
[64 bytes]   bindingSig (if spends > 0 OR outputs > 0)

// SPEND DESCRIPTION (384 bytes)

OffsetSizeFieldNotes
+032cvValue commitment
+3232anchorMerkle root
+6432nullifierEXTRACT THIS for spend detection
+9632rkRandomized public key
+128192zkproofGroth16 proof (π_A + π_B + π_C)
+32064spendAuthSigSignature
Total: 32 + 32 + 32 + 32 + 192 + 64 = 384 bytes per spend

// OUTPUT DESCRIPTION (948 bytes)

OffsetSizeFieldNotes
+032cvValue commitment
+3232cmuNOTE COMMITMENT - EXTRACT!
+6432ephemeralKeyEXTRACT for decryption
+96580encCiphertextEXTRACT - encrypted note
+67680outCiphertextOutput ciphertext
+756192zkproofGroth16 proof
Total: 32 + 32 + 32 + 580 + 80 + 192 = 948 bytes per output
CRITICAL: The encCiphertext (580 bytes) contains the encrypted note plaintext needed for trial decryption!

PITFALL: readCompactSize returns TWO values: (count, bytesConsumed)
You MUST add bytesConsumed to your offset! 
// Swift implementation
let (count, bytesRead) = readCompactSize(data, at: pos)
pos += bytesRead  // Don't forget this!

// Then process 'count' elements
for _ in 0..<Int(count) {
    pos += ELEMENT_SIZE
}

// STEP-BY-STEP PARSING GUIDE

Example: TX with 0 spends, 2 outputs

pos = 0:   header (4 bytes)        → 0x04000080
pos = 4:   versionId (4 bytes)     → 0x892F2085
pos = 8:   vin count (CompactSize) → 0x00 (0 inputs)
pos = 9:   vout count (CompactSize)→ 0x00 (0 outputs)
pos = 10:  nLockTime (4 bytes)
pos = 14:  nExpiryHeight (4 bytes)
pos = 18:  valueBalance (8 bytes)
pos = 26:  vShieldedSpend count    → 0x00 (0 spends)
pos = 27:  vShieldedOutput count   → 0x02 (2 outputs)
pos = 28:  OutputDescription[0]    → 948 bytes
pos = 976: OutputDescription[1]    → 948 bytes
pos = 1924: vjoinsplit count       → 0x00
pos = 1925: bindingSig             → 64 bytes
pos = 1989: END OF TRANSACTION

Extracting Data from OutputDescription

// For each output at position 'pos':
pos += 32  // Skip cv

let cmu = data[pos..<pos+32]  // EXTRACT
pos += 32

let ephemeralKey = data[pos..<pos+32]  // EXTRACT
pos += 32

let encCiphertext = data[pos..<pos+580]  // EXTRACT
pos += 580

pos += 80   // Skip outCiphertext
pos += 192  // Skip zkproof

// DEBUG CHECKLIST

When parsing returns empty arrays:
  1. Verify header = 0x04000080 (v4 + overwintered)
  2. Verify versionGroupId = 0x892F2085 (Sapling)
  3. Check CompactSize consumption (are you adding bytesRead?)
  4. Verify offset after each section matches expected
  5. Check data.count - is there enough data remaining?
  6. Look for zero counts that are valid (spendCount = 0 is common)

Add Debug Logging

print("📊 Spend count: \(spendCount), offset after: \(pos)")
print("📊 Output count: \(outputCount), offset after: \(pos)")
print("📊 Data remaining: \(data.count - pos) bytes")
